# Project Plan: Olist E-Commerce Analysis

## Phase 1: Data Understanding ✅
- Downloaded dataset from Kaggle (Brazilian E-Commerce Public Dataset by Olist)
- Analyzed ERD diagram and table relationships
- Created handwritten schema notes
- Defined 3 core business questions

## Phase 2: Data Preparation ✅
- Built master dataset by joining 6 relational tables (orders, order_items, customers, products, sellers, reviews)
- Cleaned and validated data, handled missing values and outliers
- Filtered to delivered orders within Jan 2017 - Aug 2018 scope
- Created calculated metrics (year_month, revenue aggregations)

## Phase 3: Analysis Modules ✅
1. **Sales Performance Analysis**: Monthly revenue trends, seasonality, KPIs
2. **Customer Geographic Analysis**: State-level customer and revenue distribution
3. **Product & Seller Analysis**: Category revenue, review scores, seller strategies, price vs. satisfaction

## Phase 4: Visualization & Reporting ✅
- Built **interactive Streamlit dashboard** with 5 pages
- Deployed dashboard to **Streamlit Cloud** for public access
- Documented findings and business recommendations
- Live dashboard: [olist-ecommerce-analytics-by-aarohi.streamlit.app](https://olist-ecommerce-analytics-by-aarohi.streamlit.app)

## 📊 Tech Stack Used
- **Python**: Pandas, NumPy for data wrangling
- **Streamlit**: interactive web app framework
- **Plotly**: interactive charts and visualizations
- **Matplotlib & Seaborn**: exploratory analysis in notebooks
- **GitHub + Streamlit Cloud**: version control and deployment
